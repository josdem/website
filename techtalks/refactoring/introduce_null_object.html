Introduce Null Object

You have repeated checks for a null value.
Replace the null value with a null object.

The essence of polymorphism is that instead of asking an object what type it is and then invoking some behavior based on the answer, you just invoke the behavior.

Don't return null, because if we do it we are essentially creating work for ourselves and foisting problems upon our callers. All it takes is one missing null check to send an application spinning out of control. If you are temped to return null from a method, consider throwing an exception or returning an special case. ~ Robert C. Martin.
